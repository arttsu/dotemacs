### **Project Specification: `create-gtd-project` (Final Version)**

**Objective:**

Create a streamlined, interactive Emacs Lisp function named `my/create-gtd-project`. This function will guide the user through a minimal set of prompts to quickly create a new Org mode project file from a template. It will leverage the user's pre-defined path constants for project directories.

---

### **User Workflow**

1.  The user executes the command via `M-x my/create-gtd-project`.
2.  **Context Prompt:** Emacs asks for the context: `Context:`. "Personal" is pre-filled, so the user can just press `RET` to accept it, or type "Open" to switch.
3.  **Title Prompt:** Emacs asks for the project title: `Project Title:`. This is a required field.
4.  **Category Prompt:** Emacs asks for the project category: `Category:`. This is a required field.
5.  **Priority Prompt:** Emacs asks for a priority: `Priority [A-E] (or RET for default '#D'):`. The user can press a single key (`A`-`E`) or `RET`.
6.  **Result:** The function automatically creates the corresponding `.org` file in the correct directory, populates it with the provided information, and opens the new file for editing.

---

### **Target Org Template & Data Mapping**

The final output file will be generated based on the following template structure. Placeholders like `<PLACEHOLDER>` show where the dynamic data will be inserted.

```org
* <PRIORITY> <TITLE> :PROJECT:
:PROPERTIES:
:ID:       <ORG_ID>
:CATEGORY: <CATEGORY>
:END:
CREATED: <TIMESTAMP> \\
AREA: AREA \\
** Resources
** TODOs
:PROPERTIES:
:STYLE:    checklist
:END:
** Log
```

Data Mapping:

<PRIORITY>: The selected priority, e.g., [#A]. Defaults to [#D].

<TITLE>: The project title provided by the user.

<ORG_ID>: A unique Org ID generated by (org-id-get-create).

<CATEGORY>: The category string provided by the user.

<TIMESTAMP>: The creation date, formatted as YYYY-MM-DD.

AREA: AREA \\: This line is a static placeholder and does not require a prompt.
